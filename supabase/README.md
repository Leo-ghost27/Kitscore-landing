# Supabase schema tracking

## Why this folder exists (read this first)

As of July 4, 2026, this repository had **zero SQL migration files, ever, in its entire git history** — every database change (functions, triggers, RLS policies, grants) across every session on this project was applied directly to Supabase, and never committed to git. This was flagged by one session as "git/production drift" — the theory being that Vercel was somehow running code that didn't come from a `git push`.

That specific theory doesn't hold up: there's no evidence anywhere (git history, branches, reflog) of frontend/API code reaching production outside of a normal push. What's actually true is narrower and less alarming: **the database schema itself was never version-controlled at all**, by any session, from the start of this project. A function existing in Supabase but not in the git repo isn't a mystery deploy — it's the expected result of applying SQL directly via the Supabase dashboard/API instead of committing a migration file, which is what every session (including this one) has been doing.

`schema-baseline-2026-07-04.sql` is a full, verified snapshot of every custom function, trigger, and RLS policy that exists in the database as of this date — generated by querying Postgres's own catalogs (`pg_proc`, `pg_trigger`, `pg_policy`) directly, not reconstructed from memory or old commit messages. It's a **reference snapshot**, not something meant to be re-run — running it against the live database would just recreate what's already there (most statements use `CREATE OR REPLACE` or `IF NOT EXISTS` patterns where the original did, but this wasn't written as an idempotent migration script, it's a documentation snapshot).

## The practice going forward

Any future database change — a new function, trigger, RLS policy, or grant — should be added as a new dated file in this folder (`YYYY-MM-DD-short-description.sql`), committed in the same push as any related application code. This is the only way to keep git and the database schema from silently diverging again.
